# NPR-OS Sandbox

**Water = informatie = energie.**
Drie perspectieven op één veld. Geen van de drie bestaat operationeel los van de andere.

---

## Bronstructuur

De sandbox onderscheidt drie lezingen van één samenhangend veld:

```
water
informatie
energie
```

Dit zijn geen drie losstaande substanties en ook geen lineaire keten waarin één component de andere twee veroorzaakt.

```
water ↔ informatie ↔ energie
```

Iedere component verschijnt alleen in verhouding tot de andere twee:

* **water** is het medium waarin verhouding, stroming en vorm kunnen verschijnen;
* **informatie** is het onderscheid en de ordening binnen dat medium;
* **energie** is de waarneembare verandering van die ordening.

Daarom geldt:

```
energie zonder medium    = ongedefinieerd
energie zonder onderscheid = onmeetbaar
informatie zonder medium = zonder positie
water zonder informatie  = ongedifferentieerd potentieel
```

Geen van de drie kan afzonderlijk als volledige oorsprong worden genomen.

De oorsprong ligt niet in één component, maar in hun onderlinge verhouding:

```
bronstructuur
:= verhouding(water, informatie, energie)
```

---

## Energie Is Geen Volledige Oorsprong

Veel hedendaagse wetenschappelijke modellen nemen energie als de meest fundamentele beschrijfbare grootheid. Materie, beweging, velden en veranderingen worden vervolgens in energetische termen uitgedrukt.

Dat is geldig binnen de meetlaag, maar niet voldoende als oorsprongsmeetkunde.

Energie beschrijft verandering. Om verandering te kunnen meten, moeten echter al aanwezig zijn:

```
een medium          waarin verschil kan verschijnen;
informatie          waardoor toestanden onderscheidbaar zijn;
een verhouding      tussen een eerdere en een latere toestand.
```

Energie kan daarom niet zonder meer de oorsprong van medium en informatie verklaren. De energiebeschrijving veronderstelt reeds de geometrie waarbinnen verandering betekenis krijgt.

```
medium + onderscheid + verhouding
→ meetbare verandering
→ energie
```

Niet:

```
energie
→ medium + onderscheid + verhouding
```

Wanneer energie toch als primaire oorsprong wordt genomen, wordt een afgeleide meetgrootheid gebruikt om de voorwaarden van haar eigen meetbaarheid te verklaren.

Binnen NPR-OS is dat een categoriefout:

```
energie     = projectie van verandering
informatie  = structuur van verschil
water       = dragend medium van verhouding
```

De bron is niet energie alleen, maar de niet-losmaakbare samenhang van alle drie.

> Wetenschap meet vooral de energetische projectie van het veld. Dat maakt energie zeer precies beschrijfbaar, maar niet automatisch tot oorsprong van het medium en de informatie die iedere energiemeting al veronderstelt.

---

## Oorsprongsmeetkunde

Een valide oorsprongsmeetkunde moet niet alleen beschrijven hoeveel verandering optreedt. Zij moet ook verklaren:

```
waar            verschil kan bestaan;
hoe             verschil wordt onderscheiden;
waardoor        verschil kan veranderen;
hoe             samenhang tijdens verandering behouden blijft.
```

Energie beantwoordt hoofdzakelijk de derde vraag: de meetbare capaciteit of toestand van verandering.

Zij levert op zichzelf niet het medium, de positie, het onderscheid, de relatie, of de grens waarbinnen een meting mogelijk wordt.

Daarom is energie geen volledige meetkunde.

Binnen de sandbox ontstaat meetkunde uit de gelijktijdige verhouding:

```
water      → continuïteit en draagvlak
informatie → onderscheid en positie
energie    → transformatie en beweging
```

Samen vormen zij een meetbaar veld:

```
geometrie :=
  continuïteit
  + onderscheid
  + transformatie
```

Geen afzonderlijke term is de oorsprong. De verhouding is primair; de termen zijn perspectieven op die verhouding.

## Talstelsels binnen de Return-Lus

Hexadecimale en decimale representaties staan niet los van elkaar.

Zij zijn verschillende projecties binnen één rekenkundige lus:

```
0 = 1
→ differentiatie
→ representatie
→ bewerking
→ reductie
→ return
→ 0 = 1
```

De primaire NPR-router gebruikt hex-native bewerkingen omdat deze route direct is:

```
C1A → 1A
```

Een decimale route is eveneens mogelijk, maar bevat extra projectiestappen:

```
C1A_hex
→ 3098_dec
→ reductie binnen de gekozen decimale functie
→ terugvertaling
→ NPR-return
```

Een decimale tussenstap is daarom niet op zichzelf fout.

Zij wordt alleen fout wanneer:

* de actieve functie niet wordt benoemd;
* `dr_dec` wordt voorgesteld als `dr_hex`;
* de tussenrepresentatie als zelfstandige oorsprong wordt behandeld;
* de route niet meer naar de gemeenschappelijke bronstructuur terugkeert.

Daarom geldt:

```
dr_hex(x) ≠ dr_dec(value(x))
```

binnen de gedifferentieerde route.

Maar beide kunnen deel zijn van dezelfde volledige lus:

```
hex-projectie
→ decimale projectie
→ reductie
→ terugkeer
≡ bronidentiteit
```

De afzonderlijke stappen blijven onderscheiden.

De volledige route sluit hun onderscheid opnieuw:

```
route + return = 0 = 1
```

`0 = 1` staat dus niet alleen vóór de route.

Het is tegelijk:

* de bron waaruit differentiatie verschijnt;
* de samenhang die tijdens de route behouden blijft;
* het return-punt waarin de volledige reeks stappen weer één wordt.

**Normatieve regel:**
> Een decimale tussenstap mag de actieve hex-functie niet ongemerkt vervangen. Zij mag wel als expliciete projectieroute worden gebruikt, mits functie, basis en return naar de gemeenschappelijke bronstructuur behouden blijven.
