### `c` — Invariant Kader

**Buiten NPR-OS:** `c` is de natuurkundige lichtsnelheid (≈ 3×10⁸ m/s).

```
medium → veldverhouding → ruimte+tijd → gemeten c
```

Niet:
```
object beweegt → nadert externe snelheidsmuur c
```

Maar:
```
de meetlaag verschijnt → haar invariant wordt als c waargenomen
```

`c` is geen limiet die beweging tegenhoudt. `c` is wat je ziet wanneer het medium als ruimte en tijd wordt gemeten.

**Maxwell als structurele aanwijzing:**

Binnen NPR-OS wordt dit gelezen als:
```
medium → veldverhouding → ruimte+tijd → gemeten c
```

`c` is daarom niet primair een grens waartegen beweging botst.
Het is de zichtbare verhouding van het veld zodra dat veld als afstand en tijd wordt gemeten.
"Sneller dan c" vormt geen externe route, omdat snelheid zelf pas binnen deze meetlaag ontstaat.

Maxwell levert de veldstructuur. NPR-OS geeft daaraan een bronlaaginterpretatie.

**Schaal-invariantie:**
```
zandkorrel ⟷ kosmos
```

Niet omdat ze even groot zijn, maar omdat dezelfde **verhouding en structuur** op beide schalen verschijnt.

- **`c`** = de invariante verhouding, zichtbaar zodra het medium als ruimte+tijd wordt gemeten
- **Planck** = kleinste betekenisvolle stap binnen dat kader
- **Kosmos** = grootste zichtbare uitdrukking ervan
- Kleinste en grootste = **twee schaalposities van dezelfde verhouding**

```
Planck ≠ kosmos          (qua grootte)

Planck  → SOURCE_0
kosmos  → SOURCE_0

**Geen ratio is het punt.** Niet `ratio(Planck) ≡ ratio(kosmos)`, maar: beide verwijzen naar `SOURCE_0`.

**"Sneller dan c"** is geen route buiten de grens, omdat snelheid zelf pas ontstaat in de laag waarin `c` verschijnt.
De limiet is geen muur vóór het veld, maar een eigenschap van de projectie van dat veld.

### `≐`, `↔`, `=`, `≠`, `≡` — Operator-Scheiding

**Vijf operatoren, vier posities, twee dwarsfuncties.**

```

**Dwarsfunctie-operatoren (door/over `0.0.0.0`):**
```

**Framelaag (↔):**
```

De framelaag beschrijft de relatie tussen bron en eerste eenheid.
`↔` betekent: wederkerige projectie tussen posities (niet gelijkheid, niet identiteit).

**Tussendoor: `↔` is geen operator binnen één positie.**
Het is de relatie TUSSEN posities — de brug van bron naar inhoud.

### Taal als brug tussen bronrekenkunde en inhoudsrekenkunde

Taal is het mechanisme dat bronidentiteit (`≐`) omzet naar
inhoudsrelaties (`=`, `≡`).

```
0 ≐ 1          bronidentiteit (≐ — ongedifferentieerd)
  → naam       taal benoemt de eenheid
  → 1 = 1      inhoudsrelatie (= — gedifferentieerd)
```

**Twee functies:**

1. **Bronbenoeming:**
   - `0 ≐ 1` is *vóór* taal

### Epistemische Herkomst: Bewuste Routing vs. Blinde Aggregatie

- welke betekenis bij een vraag past
- welke route waarschijnlijk volgt
```

**Bewuste routing:**
```
bron → context → transformatie → uitkomst
```

Hier is terug te zien:
- welke bron is gebruikt;
- welke bewerking is toegepast;
- welke aannames actief waren;
- waarom een bepaalde uitkomst verscheen.

**Blinde aggregatie (cloudmodel):**
```

```

```

een projectie is van niet meer traceerbare bronnen.

```

```

**De bottleneck:**
