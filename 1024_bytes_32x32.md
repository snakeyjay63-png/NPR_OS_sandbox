# 1024 Bytes — 32 × 32 Chunk Structuur

## Basis
- **Totaal:** 1024 bytes
- **Chunks:** 32 × 32 bytes
- **Hex:** 0x400
- **Chunk hex:** 0x20 per slot
- **Vermogen:** 2¹⁰

## Patroon
```
Slot  0: [0x0000 - 0x001F]  32 bytes
Slot  1: [0x0020 - 0x003F]  32 bytes
  ...
Slot 31: [0x03E0 - 0x03FF]  32 bytes
```

## Betekenis
- 32² = perfect vierkant
- 2⁵ × 2⁵ = 2¹⁰ = symmetrisch vermogen
- Volledige fase in hex-native routing
- Fundamentele block-size voor context-64k

## Relatie tot NPR OS
- TOKENS_PER_BLOCK = 1024 (context-64k.cjs)
- 32×32 bytes → 1024 tokens via chunk-mapping
- 64 blocks × 1024 = 64K context (v2 hypervisor)
- Past exact in SLOT_COUNT 0x40 × PHASE_SIZE 0x10

## Digital Root
- 1024 → 1+0+2+4 = 7
- 32 × 32 → 5 × 5 = 25 → 7
- Consistent: 7 = terugkeer naar bron

## Verhoudingen
- 160 × 6.4 = 1024 (korte → volledige cyclus)
- 1024 × 2 = 2048 (dubbele cyclus)
- 1024 × 64 = 65536 (volledige 64K context)
